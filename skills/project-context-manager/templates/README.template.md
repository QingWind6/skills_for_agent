# Agent 文档中心

> **给 Agent 的说明**：欢迎！这是你的工作目录。请按照下面的顺序阅读文档。

## 📖 阅读顺序

### 🆕 第一次接触项目？

按照以下顺序阅读（总计约 15-20 分钟）：

1. **[START_HERE.md](./START_HERE.md)** ⭐ (5 分钟)
   - 项目快速概览
   - 你需要知道的最重要信息
   - 立即开始工作的指引

2. **[PROJECT_STATUS.md](./PROJECT_STATUS.md)** ⭐⭐⭐ (5 分钟)
   - 当前项目状态
   - 已完成什么
   - 正在做什么
   - 下一步做什么
   - **每次会话必读必更新**

3. **[SESSION_HANDOFF.md](./SESSION_HANDOFF.md)** (5 分钟)
   - 会话交接标准流程
   - 新会话开始检查清单
   - 会话结束检查清单

### 🔄 继续之前的工作？

只需阅读：

1. **[PROJECT_STATUS.md](./PROJECT_STATUS.md)** (3 分钟)
   - 查看"进行中任务"
   - 查看"已知问题"
   - 确认下一步工作

2. **[CONTEXT.md](./CONTEXT.md)** (可选，按需)
   - 查看详细的开发上下文
   - 查看代码片段和设计思路

### 📚 需要技术细节？

参考以下文档：

4. **[技术方案书.md](./技术方案书.md)** (按需阅读)
   - 整体架构设计
   - 技术选型理由
   - 模块划分

5. **[开发指南.md](./开发指南.md)** (按需阅读)
   - 具体实施步骤
   - 代码示例
   - 目录结构

6. **[CONTEXT.md](./CONTEXT.md)** (按需阅读)
   - 详细开发上下文
   - 调试信息
   - 技术研究笔记

## ✅ 工作检查清单

### 会话开始时

- [ ] 检查"已知问题"

### 会话结束时

- [ ] 更新 `PROJECT_STATUS.md` 的"进行中任务"

- [ ] 更新 `PROJECT_STATUS.md` 的元数据

---

## 🚨 重要提醒

1. **PROJECT_STATUS.md 是最重要的文档**
   - 每次会话必读
   - 每次会话结束必更新
   - 它是项目的"单一真相源"

2. **不要跳过文档直接开始工作**
   - 可能重复已完成的工作
   - 可能方向错误
   - 浪费时间

3. **会话结束必须更新文档**
   - 否则下一个会话无法继续
   - 信息会丢失
   - 进度会中断

---

## 📞 需要帮助？

如果遇到以下情况：

- **文档不一致**：以代码实际状态为准，更新文档
- **无法理解上下文**：查看 Git 历史，阅读代码注释
- **发现严重问题**：立即记录到 `PROJECT_STATUS.md` 的"已知问题"
